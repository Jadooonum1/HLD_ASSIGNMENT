Distributed cache — consistent hashing

Hash Ring

MD5(prefix)→uint32 on a circle. First node clockwise owns the key. ~150 virtual nodes/node → even spread (~25% each over 4 nodes). Adding/removing a node remaps only ~1/N keys.

cache-node-0..3

Each = in-memory map with TTL (30s) expiry + size cap (FIFO evict). GET /cache/debug?prefix= shows owner + hit/miss.

Cache invalidation

Drop cached results for every prefix of the query, so the next suggest recomputes.

batch

Batch Writer

Buffer + aggregate duplicates in memory. Flush by size (500) or timer (2s) → ~99% fewer writes. (Crash-before-flush trade-off discussed in DESIGN.md.)
